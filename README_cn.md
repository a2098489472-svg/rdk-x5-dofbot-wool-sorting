# 基于 RDK X5 的羊毛绒异纤分拣机械臂设计

## 一、项目简介

本项目面向羊毛绒及纤维类物料中的异纤杂质分拣任务，设计并实现了一套基于 RDK X5、USB 摄像头和 DOFBOT SE 六自由度机械臂的视觉引导智能分拣系统。系统通过摄像头采集作业区域图像，利用目标检测算法识别羊毛绒中的树叶、纸屑、塑料片等异纤目标，并结合坐标转换、路径规划和机械臂控制方法，实现异纤目标的自动识别、定位、抓取与分拣。

本项目采用“上位机视觉决策 + 下位机机械臂执行”的系统架构。其中，RDK X5 作为主控计算平台，负责图像采集、目标检测、坐标转换、路径规划和控制指令生成；DOFBOT SE 机械臂及其 STM32 舵机控制板作为执行端，负责接收控制指令并驱动机械臂各关节舵机完成抓取动作。系统能够实现从视觉感知到机械执行的闭环控制，适用于嵌入式视觉抓取、智能分拣和机械臂教学验证等应用场景。

## 二、项目功能

本项目主要实现以下功能：

1. USB 摄像头图像采集；
2. 羊毛绒异纤目标检测与识别；
3. 异纤目标像素坐标提取；
4. 图像坐标到机械臂坐标的转换；
5. 机械臂抓取目标位置计算；
6. 机械臂抓取、抬升、转移和释放控制；
7. DOFBOT SE 舵机控制；
8. 异纤目标自动分拣执行；
9. 支持 RDK X5 嵌入式端部署运行；
10. 支持 Docker 容器环境下运行调试。

## 三、系统总体架构

系统整体由视觉感知模块、目标定位模块、路径规划模块和机械臂执行模块组成。

系统工作流程如下：

1. USB 摄像头采集羊毛绒分拣区域图像；
2. RDK X5 获取实时图像数据；
3. 目标检测模型识别图像中的异纤目标；
4. 系统提取目标类别、置信度和像素坐标；
5. 根据相机标定与坐标转换关系，将图像坐标转换为机械臂可执行坐标；
6. 根据目标位置生成机械臂运动控制指令；
7. RDK X5 通过串口/USB 通信方式向 DOFBOT SE 舵机控制板发送控制指令；
8. STM32 舵机控制板输出 PWM 信号驱动机械臂各关节舵机；
9. 机械臂完成目标抓取、转移和释放，实现异纤分拣。

## 七、运行环境与启动方式

本项目运行于 RDK X5 的 Docker 容器环境中。运行前需要确保摄像头、机械臂、电源和通信接口连接正常。

### 1. 启动 Docker 容器

在 RDK X5 终端中执行：

```bash
docker start dofbot_noetic
docker exec -it dofbot_noetic bash
```

### 2. 加载 ROS 环境

进入 Docker 容器后执行：

```bash
source /opt/ros/noetic/setup.bash
source /home/yahboom/dofbot_ws/devel/setup.bash
```

### 3. 进入代码目录

根据实际部署路径进入代码所在目录，例如：

```bash
cd /home/yahboom/dofbot_ws/src/dofbot_color_identify/scripts
```

### 4. 运行主程序

执行：

```bash
python3 main_code8.py
```

程序启动后，可根据终端提示输入目标类别或控制命令，实现目标检测、定位、路径规划和机械臂抓取执行。

## 八、系统运行流程

系统启动后，主要运行流程如下：

1. 初始化摄像头；
2. 初始化机械臂控制接口；
3. 读取摄像头图像；
4. 调用目标检测模型识别异纤目标；
5. 获取目标中心点坐标；
6. 根据标定关系完成坐标转换；
7. 生成机械臂抓取位置；
8. 控制机械臂移动至目标上方；
9. 控制机械臂下降到抓取高度；
10. 控制夹爪闭合完成抓取；
11. 控制机械臂抬升并移动到放置区域；
12. 控制夹爪张开完成释放；
13. 返回初始或观察姿态，等待下一次任务。

## 九、主要控制逻辑

系统采用视觉引导方式完成异纤抓取任务。主控端通过摄像头获取图像后，检测模型输出异纤目标的类别、置信度和位置信息。系统根据目标框中心点计算目标像素坐标，并通过相机标定和坐标映射方法转换为机械臂工作空间坐标。

机械臂控制部分根据转换后的目标坐标，按照“预抓取—下降—夹取—抬升—转移—释放”的动作顺序执行。具体流程如下：

1. 夹爪打开；
2. 机械臂移动至目标上方安全位置；
3. 机械臂下降至抓取高度；
4. 夹爪闭合抓取目标；
5. 机械臂抬升；
6. 机械臂移动至放置区域；
7. 夹爪打开释放目标；
8. 机械臂返回观察姿态。

## 十、通信方式

RDK X5 与 DOFBOT SE 舵机控制板之间主要通过串口/USB 方式进行通信。通信关系如下：

| 连接关系 | 功能 |
| --- | --- |
| RDK X5 TX → DOFBOT SE RX | 主控板向机械臂控制板发送控制数据 |
| RDK X5 RX ← DOFBOT SE TX | 控制板向主控板反馈状态信息 |
| GND ↔ GND | 保证通信参考地一致 |
| STM32 → S1-S6 舵机接口 | 输出 PWM 信号控制各关节舵机 |

供电方面，RDK X5 主控板和机械臂舵机系统采用独立供电方式：

| 模块 | 供电方式 |
| --- | --- |
| RDK X5 主控板 | 5V / USB-C 供电 |
| DOFBOT SE 舵机控制板 | 6～8V 独立舵机电源 |
| USB 摄像头 | 由 RDK X5 USB 接口供电 |
| 舵机 | 由舵机电源模块供电 |

独立供电可以减少舵机大电流动作对主控板造成的干扰，提高系统运行稳定性。

## 十三、注意事项

1. 运行代码前，需要确保 RDK X5 已正确连接 USB 摄像头；
2. 运行代码前，需要确保 DOFBOT SE 机械臂供电正常；
3. 舵机供电建议采用独立电源，不建议直接由主控板供电；
4. 运行前需要确认 Docker 容器 `dofbot_noetic` 已正确配置；
5. 运行前需要加载 ROS 环境；
6. 若摄像头无法打开，需要检查 `/dev/video0` 或 `/dev/video1` 设备节点；
7. 若机械臂无动作，需要检查串口连接、舵机供电和机械臂控制库是否正常；
8. 本仓库代码为核心功能代码，完整运行依赖实际硬件环境和本地部署配置。
